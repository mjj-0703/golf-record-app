# 要件定義 — ゴルフ感覚メモアプリ

> 本ドキュメントは開発方針・要件のソースオブトゥルース（SoT）です。  
> 実装状況: MVP v1（工程1〜11 完了）

## 1. 概要

### 1.1 アプリ名

ゴルフ感覚メモ

### 1.2 目的

ゴルフの練習・ラウンド時に感じた **「良かった感覚」** と **「ミスの原因」** を記録し、次回プレー時に忘れないようにする。

### 1.3 ターゲットユーザー

- ゴルフ初心者〜100切りレベル
- 記録を習慣化したいが、入力の手間に弱い層

### 1.4 成功基準（MVP）

- 練習・ラウンド後、短時間で記録を残せる
- 過去の記録を一覧・詳細で見返せる
- アプリ再起動後も記録が残る
- スマートフォン（Android）で動作する

## 3. 画面要件

### 3.1 記録一覧画面（`RecordListPage`）

**役割:** 保存済み記録の確認・フィルタ・新規作成への導線

| 要素 | 仕様 |
|------|------|
| AppBar | タイトル「ゴルフ感覚メモ」 |
| フィルタ | 種別、タグ（6種）、ウッド/アイアン番手、アプローチ種類 |
| 一覧 | カード形式、日付降順 |
| 空状態 | アイコン + 説明文 |
| FAB | `+` で新規作成画面へ遷移 |
| 戻るボタン | なし（ホーム画面） |

### 3.2 記録作成 / 編集画面（`RecordFormPage`）

**役割:** 記録の入力・更新

| 項目 | 必須 | 入力方式 |
|------|------|----------|
| 日付 | ✅ | DatePicker |
| 種別（練習 / ラウンド） | ✅ | SegmentedButton |
| 良かった感覚 | ✅ | 複数行テキスト |
| ミスの原因 | ✅ | 複数行テキスト |
| 次回試すこと | ✅ | 複数行テキスト |
| クラブ | 必須（1つのみ） | ドライバー / ウッド / アイアン / アプローチ / パター |
| ウッド番手 | ウッド選択時必須 | 3W / 5W / 7W |
| アイアン番手 | アイアン選択時必須 | 3〜9 |
| アプローチ | アプローチ選択時必須 | PW / SW / ロフト角（48°〜60°） |
| メンタル | 任意（クラブに追加可能） | 単独の2つ目タグとしてのみ |
| メモ | 任意 | 複数行テキスト |

- 保存時: 必須3項目が空ならエラー表示、保存不可
- 編集時: 既存値をフォームにプリセット
- 戻る: AppBar の ← または Android システム戻る

### 3.3 記録詳細画面（`RecordDetailPage`）

**役割:** 1件の記録を見返す

- 全項目をカード形式で表示
- 編集ボタン → 作成/編集画面（編集モード）
- 削除ボタン → 確認ダイアログ → 削除 → 一覧へ戻る

## 4. データ要件

### 4.1 Record エンティティ

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | String | 一意識別子（新規作成時は microsecond タイムスタンプ） |
| `date` | DateTime | 記録対象日 |
| `type` | SessionType | `practice`（練習）/ `round`（ラウンド） |
| `goodFeel` | String | 良かった感覚 |
| `missCause` | String | ミスの原因 |
| `nextTry` | String | 次回試すこと |
| `tags` | List\<String\> | タグ（0件以上） |
| `memo` | String | 自由メモ |
| `createdAt` | DateTime | 作成日時 |
| `updatedAt` | DateTime | 更新日時 |

### 4.2 固定タグ一覧

`ドライバー` / `ウッド` / `アイアン` / `アプローチ` / `パター` / `メンタル`

**詳細タグ（保存形式）:**

| 親タグ | 選択 | 保存例 |
|--------|------|--------|
| ウッド | 3W / 5W / 7W | `5番ウッド` |
| アイアン | 3〜9 | `7番アイアン` |
| アプローチ | PW / SW / ロフト角 | `52°アプローチ` |
| メンタル | — | `メンタル`（メインクラブに追加） |

1記録あたりメインクラブは1つのみ。メンタルのみ2つ目として追加可能。

ドライバー・パター・メンタルは詳細選択なし。旧データの `アイアン` / `アプローチ` 等もフィルタ・表示で互換対応。

### 4.3 永続化

- ストレージ: `shared_preferences`
- キー: `records_v1`
- 形式: Record 配列の JSON 文字列
- ソート: 読み込み時・保存時に `date` 降順

## 5. バリデーション

| 項目 | ルール |
|------|--------|
| 良かった感覚 | 空文字不可 |
| ミスの原因 | 空文字不可 |
| 次回試すこと | 空文字不可 |
| メモ | 任意（空可） |
| タグ | 任意（0件可） |
| ウッド番手 | ウッド選択時は必須 |
| アイアン番手 | アイアン選択時は必須 |
| アプローチ種類 | アプローチ選択時は必須 |

> 文字数上限は現状未設定。必要に応じて将来追加。
